Explorer Recommender RAG

About

Explorer Recommender RAG is an Ensemble Retrieval and RAG Application with an integrated Recommender System, utilizing Google Local Data (2021).

Model Description

Retrieval

For retrieval, we make use of ensemble retrieval which combines two different retrieval systems and reranks the documents returned based on the weightage given to each system.

We utilize BM25 and vector-based embedding retrieval systems. A document in our use case is a combination of restaurant name, description, service options, ratings, and all the user reviews. For embedding retrieval systems, we pre-compute the embedding vector for each restaurant's document, which are stored in a vector store using Chroma DB. While retrieving, we obtain 10 documents from BM25 based on a query and another 10 from the vector store. The 10 retrieved from the vector store are selected based on similarity with the embedding vector of the query.

Subsequently, the ensemble retrieval system reranks those documents (restaurants) based on the weightage given to the underlying systems. Equal weightage is assigned to both BM25 and Embedding retrieval system.

Recommendation Model

We have used both item-item collaborative filtering and content-based similarity between two items:

ITEM-ITEM Collaborative Filtering:

We have the user-item ratings, recalculated the ratings of the user-item pair based on the rating and sentiment polarity of the review text, and built a matrix of user-item ratings.

Calculated the Item-Item similarity based on Pearson similarity and created an ITEM-ITEM similarity matrix.

Content-based Filtering:

We have created an item-item similarity based on the content of each restaurant details such as Description, Category options available, Service Options, Accessibility options, and Reviews.

Using the Bert-based embedding model, we have found the cosine similarity between the two items and created an ITEM-ITEM similarity matrix for all the items.

Based on the above 2 similarity matrices, we have created a new similarity matrix containing the top 5 neighbors of each item, which are the most similar ones.

AI Generated Review Summarization model:

We have used the Langchain LLM model to summarize all the reviews generated by the users for a particular restaurant into a neutral summary, incorporating both positive and negative reviews. For Retrieval, we make use of ensemble retrieval, combining two different retrieval systems and reranking the documents returned based on weightage given to each system.

We make use of BM25 and vector-based embedding retrieval systems. A document in our use case is a combination of restaurant name, description, service options, ratings, and all the user reviews. For embedding retrieval systems, we pre-compute the embedding vector for each restaurant's document. These vectors are stored in a vector store, using Chroma DB. While retrieving, we get 10 documents from BM25 based on a query and another 10 from the vector store. The 10 retrieved from the vector store are selected based on similarity with the embedding vector of the query.

Then the ensemble retrieval system reranks those documents (restaurants) based on the weightage given to the underlying systems. Equal weightage is given to both BM25 and Embedding retrieval system. For the RAG part, we make use of the documents returned by the ensemble system. We extract only the name, description, rating, and service options from the documents, leaving out the reviews. This, along with the query, is passed to an LLM (LangChain API to define the RAG chain) with a prompt template to get the retrieval augmented generation text.

Prerequisites

  • OpenAI Key stored on your local machine
  • Environment variable 'OPENAI_API_KEY' containing OpenAI key
  • Two pickle files: restaurent_docs_1.pickle and recommendation_1.pickle
  • Vector Store Folder "chroma_db_res2/"
  • Download pickle files and Chroma DB folder from Google Drive

Getting Started

  • Install python packages from requirements.txt: pip install -r requirements.txt
  • Install Node 12
  • To start the backend server, run: python3 app.py
  • To run React, navigate to ISRFrontend Folder and execute: npm install and npm start
